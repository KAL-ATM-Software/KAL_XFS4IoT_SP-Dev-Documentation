    ErrorHandler

    The registered fatal error handler.

    Declaration

    Property Value
    Type Description
    ErrorHandling.ErrorDelegate
    Remarks

    This should be set during program setup with the required fatal error handling mechanism. This could be a real fatal error handler, or it could be something like special version for unit testing. The fatal error handler can only be set once.

    Remarks

    This version can be used in an throw statement inline with an expression. For example

    var x = y!=null ? "Some value" : throw InvokeErrorHandler<string>( "Something failed" );

    though this is normally wrapped in a contract assertion rather than being called directly.
