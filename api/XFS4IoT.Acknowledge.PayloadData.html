    Status

    "ok" if the command was successful and has been queued. The command will complete with a completion message.Otherwise there is an error that stops the command from being queued and there will be no further messages.

    • ok - The command has been accepted for execution
    • invalidMessage - The JSON in the message is invalid and can't be parsed.
    • invalidRequestID - The request ID on the command is invalid.This could be because the value was not an integer, had a zero value, or because a command with the same request ID was already queued or is executing.
    • tooManyRequests - The service has currently received and queued more requests than it can process.
